# Network Traffic Analysis Report

## Summary
Network traffic was captured and analyzed to identify potential malicious or suspicious activity within a controlled lab environment.

## Tools Used
- Wireshark
- Packet capture utilities
- Linux

## Investigation Process
1. Captured live network traffic
2. Applied filters to isolate relevant protocols
3. Analyzed packet behavior and communication patterns
4. Identified anomalies in traffic flow

## Key Findings
- Detected unusual traffic patterns from specific IP addresses
- Observed repeated connection attempts indicating possible scanning or brute-force behavior
- Identified abnormal protocol usage and packet frequency

## Conclusion
The analysis revealed indicators of suspicious activity consistent with common network-based attacks, demonstrating the ability to monitor and interpret network traffic effectively.
